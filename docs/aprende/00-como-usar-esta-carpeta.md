# Cómo usar esta carpeta

Estos apuntes existen porque este arcade no es solo "hacer juegos" — es la excusa para aprender lo
que hace falta para tenerlos corriendo de verdad, en tu propio servidor, accesibles desde internet.
Están pensados para poder entrar sin saber programar y avanzar de cero, con teoría corta + ejemplos
que existen de verdad en este mismo repo (no inventados).

Orden sugerido (cada uno da por hecho el anterior):

1. [01 – Qué son HTML, CSS y JS](01-que-son-html-css-js.md)
2. [02 – El JavaScript que necesitas](02-javascript-que-necesitas.md)
3. [03 – Qué son las librerías](03-que-son-las-librerias.md)
4. [04 – Qué es una API](04-que-es-una-api.md)
5. [05 – Qué es REST](05-que-es-rest.md)
6. [06 – Qué es CORS](06-que-es-cors.md)
7. [07 – Manejo de errores: try/catch](07-manejo-de-errores-try-catch.md)
8. [08 – Promesas y async/await](08-promesas-y-async-await.md)
9. [09 – localStorage](09-localstorage.md)
10. [10 – WebSockets vs HTTP](10-websockets-vs-http.md)
11. [11 – Arquitectura cliente-servidor de este arcade](11-arquitectura-cliente-servidor.md)
12. [12 – Docker y despliegue](12-docker-y-despliegue.md)
13. [13 – Gitflow en este repo](13-gitflow-en-este-repo.md)

No hace falta leerlos todos de una sentada. La idea es: cuando vayas a construir un juego nuevo y
te trabes con un concepto, vienes aquí, lo lees, y vuelves al código. Todos tienen ejemplos que
puedes probar en este mismo repo (consola del navegador, `curl`, o el propio `server.js`).
