# Cómo usar esta carpeta

Estos apuntes existen porque este arcade no es solo "hacer juegos" — es la excusa para aprender lo
que hace falta para tenerlos corriendo de verdad, en tu propio servidor, accesibles desde internet.

Orden sugerido (cada uno da por hecho el anterior):

1. [01 – El JavaScript que necesitas](01-javascript-que-necesitas.md)
2. [02 – Qué es una API](02-que-es-una-api.md)
3. [03 – Qué es REST](03-que-es-rest.md)
4. [04 – Qué es CORS](04-que-es-cors.md)
5. [05 – Promesas y async/await](05-promesas-y-async-await.md)
6. [06 – WebSockets vs HTTP](06-websockets-vs-http.md)
7. [07 – Arquitectura cliente-servidor de este arcade](07-arquitectura-cliente-servidor.md)
8. [08 – Docker y despliegue](08-docker-y-despliegue.md)
9. [09 – Gitflow en este repo](09-gitflow-en-este-repo.md)

No hace falta leerlos todos de una sentada. La idea es: cuando vayas a construir un juego nuevo y
te trabes con un concepto, vienes aquí, lo lees, y vuelves al código. Todos tienen ejemplos que
puedes probar en este mismo repo (consola del navegador, `curl`, o el propio `server.js`).
